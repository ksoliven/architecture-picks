Recommended Path

Build PICKS as a modular architecture around a centralized Integration Layer.

PICKS should not duplicate future ERP or MES functionality. It should coordinate logistics, lab execution, program management, governance, and external systems through a controlled integration layer so GTRI can improve traceability without forcing a single disruptive replacement.

Recommended Path

Integrate instead of replacing everything.

Keep useful existing systems, add governance and RBAC controls, modernize key workflows, and use a centralized Integration Layer to connect Oracle ERP, Workday, PEDYN, and the selected MES.

Design Priorities

Security and traceability lead the design.

The SEMP ranks security, traceability, availability, performance, and adaptability as the quality attributes that drive the architecture.

Future Pathway

Move forward in phases.

Start with MVP scoping, then plan ERP/MES integration, validate usability and pilot workflows, and expand across labs as requirements mature.

Why This Path

It matches the rescope and the final architecture evidence.

USG ERP dependency

Oracle has been identified as the preferred ERP direction, but the final NetSuite or Fusion platform choice remains dependent on organizational, cybersecurity, hosting, and integration requirements.

Lab adoption risk

Interviews showed fragmented workflows, manual transfer, and legacy usability problems. A phased modular approach lets GTRI improve high-value workflows without forcing every lab to change at once.

Compliance readiness

The SEMP emphasizes CMMC, audit logging, access control, and traceability. A governance layer gives those concerns a clear architectural home.

Known Dependencies

Several decisions outside PICKS will shape the implementation path.

USG ERP timeline

The University System of Georgia ERP rollout will affect when PICKS can validate final enterprise interfaces and complete full ERP-based V&V.

Oracle and Workday decisions

Oracle is the preferred ERP direction for procurement, vendor, inventory, and planning data, while Workday remains the financial and funding data source. Final platform choices will determine exact APIs, data fields, and synchronization rules.

MES selection

Client feedback currently favors Tulip first because of validated FedRAMP status and potential Workday integration, with Odoo second because of stronger apparent integration and a possible on-prem path. InvenTree should not advance unless its integration and security fit materially improve.

Legacy lab tool transition

PEDYN and other lab tools contain useful history, so migration or adapter planning is needed before broad rollout.
