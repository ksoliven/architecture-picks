Reference

Site Acronyms

Definitions for abbreviations that may appear in system names, course material, examples, or source links.

API

Application Programming Interface

A technical way for systems to send and receive information.

ASE

Applied Systems Engineering

Course/program context for the capstone presentation.

BDD

Block Definition Diagram

A technical diagram style for showing system parts and relationships.

BOM

Bill of Materials

Structured list of parts, components, and quantities needed for a build or product configuration.

CMMC

Cybersecurity Maturity Model Certification

Cybersecurity framework that may affect future security planning.

ERP

Enterprise Resource Planning

Business system category for purchasing, inventory, finance, and supply-chain data.

GTRI

Georgia Tech Research Institute

Organization context for the PICKS presentation.

ID

Identifier

Unique value used to identify records, transactions, messages, or system entities.

iPaaS

Integration Platform as a Service

Managed cloud connection platform category used for tools such as MuleSoft and Boomi.

MES

Manufacturing Execution System

Lab or shop-floor system category for work orders, builds, inventory use, and history tracking.

MVP

Minimum Viable Product

First-version lens focused on useful capability with low disruption and manageable complexity.

PEDYN

Existing Lab System Name

Existing lab-record source referenced for historical data and connected lab history.

PICKS

Capstone System Name

Project acronym used for the proposed inventory and connection concept.

PO

Purchase Order

Purchasing record used to track ordered materials, vendor details, quantities, and related costs.

RBAC

Role-Based Access Control

Access model that grants or blocks actions based on a user role and rules.

SEMP

Systems Engineering Management Plan

Project document that explains the systems engineering approach, planning, responsibilities, and supporting technical work.

SNR

Stakeholder Need Requirement

Requirement identifier prefix used for stakeholder-level needs before they are decomposed into system requirements.

SYS

System Requirement

Requirement identifier prefix used in course and supporting detail views.

TLS

Transport Layer Security

Security method used to protect information shared between users and systems.

USG

University System of Georgia

Georgia system direction context referenced as an input to future business-system decisions.
