Connection Choices

Ways PICKS Can Connect Systems

The Integration Layer is the architecture pattern. MuleSoft, Boomi, or a custom API gateway are implementation options for that logical connector.

MuleSoft

Best fit when the organization needs strong oversight, reusable system connections, and long-term management.

Strong oversightStrong data matchingHigher cost
Boomi

Best fit when the team needs to connect systems quickly and test work processes with less custom coding.

Fast setupVery strong data matchingEasier upkeep
Custom API Gateway

Best fit when the team needs full control and has enough technical capacity to build and maintain it.

High controlStrong securityMore upkeep

Exchange Results

What the System Reports Back

Success

The request worked. The information was found, saved, updated, or shared correctly. 200 / 201

Data Issue

Something required is missing, formatted incorrectly, or does not match the receiving system. 400 / 422

Access Blocked

The person, service account, or role is not allowed to complete this exchange. 401 / 403

Record Mismatch

The record was not found, is duplicated, or conflicts with newer information. 404 / 409

System Unavailable

One of the connected systems is down or failed while processing the exchange. 500 / 503

Information Inventory

What PICKS Sends and Receives

Step Exchange Action Information Included How PICKS Handles It Purpose Example
01 User to Access Rules
Ask to Take Action		 Ask User identity, requested action, target system Checks who the user is and whether they are allowed to take the action. Protects shared information
02 Access Rules to User
Approve or Block Action		 Reply Role match, allowed permission, session decision Gives a clear yes or no decision and keeps the decision for review history. Keeps a reviewable decision record
03 User to Connection Hub
Send Approved Request		 Send Approved request, part number, build list or work context Organizes the request before sending it to finance, ERP, or MES systems. Starts the approved work process
04 Connection Hub to Oracle ERP
Ask for Purchasing and Inventory Data		 Ask Purchasing lookup, inventory lookup, asset identifier Looks up purchasing, inventory, and asset information from the ERP system. Finds purchasing and inventory data
05 Oracle ERP to Connection Hub
Return Purchasing and Inventory Data		 Reply Purchase order, vendor, part number, quantity, inventory status Converts ERP fields into a common PICKS format before other systems use them. Turns business data into usable PICKS information
06 Connection Hub to Workday
Ask for Funding Information		 Ask Project account, cost object, funding reference Checks project and funding context without mixing finance data into lab records. Adds funding context
07 Workday to Connection Hub
Return Funding Information		 Reply Funding status, financial reference, project cost data Adds funding context to the transaction while inventory remains managed elsewhere. Keeps funding context connected to the request
08 Connection Hub to PEDYN
Ask for Existing Lab Data		 Ask Part number, build identifier, lab record lookup Finds older lab records while protecting the rest of PICKS from inconsistent older data. Finds older lab history
09 PEDYN to Connection Hub
Return Existing Lab Data		 Reply Older part record, build history, lab status Turns older lab information into a connected format that modern systems can use. Keeps older lab history connected
10 Connection Hub to MES
Create or Update Lab Record		 Send Mapped item record, inventory update, operation status, trace link Sends the cleaned record to the MES so the operational record can be created or updated. Updates lab and inventory work
11 MES to Connection Hub
Confirm Update Status		 Reply Update result, lab record ID, sharing status Confirms whether the update worked and gives PICKS a status it can track. Confirms the update result
12 Connection Hub to Activity Record
Record What Happened		 Record Transaction ID, user, systems touched, timestamp, decision outcome Saves a connected activity record for review, troubleshooting, and compliance evidence. Keeps an activity record
